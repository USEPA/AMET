<p>All 14 environment variables in this section are T/F flags.  Flags can be set to T or F depending on what post-processing files are needed and which steps have already been completed. While the flags can be set in many different permutations, the post-processing must take place in a specific order:</p>
<ol>
<li>Run the combine utility on CCTM output to create COMBINE_ACONC and COMBINE_DEP files.</li>
<li>Create "sitex" run scripts for controlling the execution of the sitecmp and sitecmp_dailyo3 utilities.</li>
<li>Run the sitex scripts to create comma separated files with matched model/obs pairs for different observation networks.</li>
<li>[Optional] Load model/obs pairs into the AMET database.</li>
<li>Create evaluation plots based on matched model/obs data using the batch plotting code in AMET.</li>
</ol>
<p>A user can choose to do all of the steps at once or run the script multiple times.  For example, the script can be set to only run combine by setting the first flag to T and the remaining flags to F.  The user can then rerun the script at a later time to create the sitecmp files and evaluation plots.  In this case the first flag can be set to F since the combine files already exist.</p>
<p><em>Notes</em></p>
<ul>
<li>A user has the option to create an AMET project and load the model/obs data into the AMET MYSQL database. Loading the data into the database allows users who have access to the RTP campus Intranet to access the data online through the <a href="http://newton.rtpnc.epa.gov/wyat/AMET_AMAD/querygen_aq.php" rel="nofollow">AMET web interface</a>.  The web interface allows for more refined control over the evaluation plots.  Loading the data into the AMET database also allows the users to evaluate the model output across all of the model/obs data in the simulation period rather than the default mode which produces monthly summaries.  This option is set in section 7.</li>
<li>AMET_DB should only be set to T if LOAD_SITEX=T (or if LOAD_SITEX has been set to T previously).</li>
<li>An AMET project does not have to be created in order to use the AMET batch plotting scripts.  If the user chooses not to load the data into the AMET database, they should set the AMET_DB flag to F.  In this case the batch plotting scripts will read the data directly from the .csv sitecmp and sitecmp_dailyo3 files.</li>
</ul>

<p><strong>Required Met and CCTM files</strong></p>
<ol>
<li><strong>METCRO2D</strong> - needed for instantaneous hourly surface temperature (TEMP2), planetary boundary height (PBL), solar radiation (RGRND), 10m wind speed (WSDP10), 10m wind direction (WDIR10), precipitation (RN+RC).</li>
<li><strong>METCRO3D</strong> - needed for instantaneous hourly air density (DENS) which is used in unit conversions of gas and aerosol species</li>
<li><strong>CCTM_ACONC</strong> - needed for hourly average gas and aerosol modeled species time stamped at the top of the hour</li>
<li><strong>CCTM_APMDIAG</strong> - needed for hourly average relative humidity (RH) and modeled aerosol mode parameters time stamped at the top of the hour</li>
<li><strong>CCTM_WETDEP1</strong> - needed for hourly summed gas and aerosol wet deposition species time stamped at the top of the hour</li>
<li><strong>CCTM_DRYDEP</strong> - needed for hourly summed gas and aerosol dry deposition species time stamped at the top of the hour</li>
</ol>
<p><em>Notes</em></p>
<ul>
<li>PM2.5 modeled size distributions from the CCTM_APMDIAG file are used to calculate PM2.5 species with a cut-off diameter of 2.5Œºm or less.  These species begin with "PM" in the species definition files provided in the CMAQ code base for version 5.2 or later. For example, PM25_NA is all sodium that falls below 2.5Œºm diameter.  These 'PM' variables are used for comparisons at IMPROVE and CSN sites.</li>
<li>Prior to CMAQv5.2, aerosol modeled size distributions were contained in the AERODIAM file which contained instantaneous hourly model variables starting with hour 1.  In CMAQv5.2 the CCTM_APMDIAG output was created to produce hourly average model variables starting with hour 0 which is analogous to the structure of the CCTM_ACONC output file.  This script is structured to <em>only</em> work with the CCTM_APMDIAG file for extracting model size distributions.  If the CCTM_APMDIAG file was not produced by the model simulation (by setting CTM_APMDIAG flag to F in the run_cctm.csh run script) then this evaluation script can be modified to remove the dependency on the CCTM_APMDIAG file.  See section 4 for more details.</li>
<li>Surface temperature and relative humidity are used to calculate an "FRM equivalent" PM2.5 total estimate that accounts for loss of particle nitrate, sulfate and ammonium from the FRM sampling filters. These species are labeled with "_FRM" in the concentration species definition files provided in the CMAQ code base for versions 5.2 and later, i.e. PMIJ_FRM and PM25_FRM.</li>
</ul>
<p><strong>Naming Conventions for Input/Output Files</strong>
Consistent naming conventions are used throughout the script to facilitate looping over dates.</p>
<ul>
<li>This script assumes MET files are dated with the following naming convention:
<pre><code>${METCRO2D_NAME}_${YY}${MM}${DD}.nc   
${METCRO3D_NAME}_${YY}${MM}${DD}.nc
</code></pre>
</li>
<li>This script assumes daily CCTM output files are dated with the following naming convention:
<pre><code>${CCTM_Name}_${YYYY}${MM}${DD}.nc
</code></pre>
For example: <em>CCTM_ACONC_v52_intel17.0_SE52BENCH_20110701.nc</em></li>
<li>This script will create monthly combine files that are dated with the following naming convention:
<pre><code>${COMBINE_ACONC_NAME}_${YYYY}${MM}.nc  
${COMBINE_DEP_NAME}_${YYYY}${MM}.nc
</code></pre>
</li>
</ul>
<p>File names can be adjusted but may require changes to the script below section 7.</p>
<p><strong>Organization of post-processing output</strong>
Output files are organized into three directories:</p>
<ul>
<li>$POSTDIR - location to write combine files, or the location of existing combine files</li>
<li>$EVALDIR - location to save sitecmp and sitecmp_dailyo3 .csv files for each network</li>
<li>$PLOTDIR - location to save evaluation plots</li>
</ul>
<p>These directories can be set to the same path.  This run script is set up to organize the various post-processing steps into monthly files.  For example, if a user has an annual simulation and uses the script to go through all of the post-processing steps the end result will include:</p>
<ol>
<li>12 monthly COMBINE_ACONC and 12 monthly COMBINE_DEP files with hourly model output, all written to the $POSTDIR directory.</li>
<li>12 .csv files with matched model/obs data for EACH network selected in Section 5.  These files will be organized into 12 directories labeled $EVALDIR/$YYYY$MM.</li>
<li>Evaluation plots for each month of model/obs pairs, organized into 12 directories labeled $PLOTDIR/$YYYY$MM.</li>
</ol>
<p>For a 2-week simulation that spans two months, e.g. 6/15/2011 - 7/15/2011, evaluation plots will still be divided into monthly summaries, using the available model data from each month. If the user would prefer the evaluation plots be based on ALL available data from the simulation time period, rather than monthly summaries, there is an option for this in section 7.</p>

<pre><code> AMET_DATABASE             AMET database name, e.g. amad_CMAQ_v52_Dev. Model to model

<h2><a id="user-content--section-8-execution-of-all-post-processing-steps" class="anchor" aria-hidden="true" href="#-section-8-execution-of-all-post-processing-steps"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a><a id="user-content-execute"></a> Section 8: Execution of all post-processing steps</h2>
<p>The user will typically not need to edit this portion of the run script.  Section 8 is divided into 6 subsections:</p>
<ul>
<li>8a -Loop through simulation days to create ACONC combine files for every month</li>
<li>8b - Loop through simulation days to create DEP combine files for every month</li>
<li>8c - Advanced AMET configuration options</li>
<li>8d - Create AMET project</li>
<li>8e - Create and run sitecmp run scripts. Load sitecmp data into AMET database.  Make plots for each month of model/obs data.</li>
<li>8f - Create evaluation plots for the entire simulation period.</li>
</ul>
<p>Looping across simulation days is controlled using the Linux <em>date</em> command which can be used for both Gregorian and Julian date formats.  A user can make change to sections 8a, 8b and/or 8e to change the looping structure or to modify the default file naming conventions described in section 2. Below are a few examples of <em>date</em> commands that a user may find helpful in customizing their script.</p>
<ul>
<li>Retrieve calendar month from date using format MM</li>
</ul>
<pre><code>date -ud "2018-06-30" +%m
06
</code></pre>
<ul>
<li>Retrieve calendar year from date using format YYYY</li>
</ul>
<pre><code>date -ud "2018-06-30" +%Y
2018
</code></pre>
<ul>
<li>Convert Gregorian date YYYY-MM-DD to Julian date YYYYJJJJ</li>
</ul>
<pre><code>date -ud "2018-06-30" +%Y%j
2018181
</code></pre>
<ul>
<li>Increment Gregorian date by one day</li>
</ul>
<pre><code>date -ud "2018-06-30+1days" +%Y-%m-%d
2018-07-01
</code></pre>
